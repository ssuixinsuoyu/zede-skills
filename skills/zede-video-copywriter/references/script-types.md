# 口播文案类型结构

只读取并使用观点卡中已确认的类型。结构服务于观点，不要为了填满模板增加无关内容。

## 知识型

适用于解释概念、机制、趋势或认知框架。

推荐结构：

1. 用问题、冲突或结果说明为什么值得听。
2. 直接给出核心观点。
3. 解释关键机制或因果链。
4. 用一个用户提供的例子、类比或事实帮助理解。
5. 回到观点，说明它改变了什么认识。

避免从定义、背景史或“今天给大家分享”开始。不要为了显得专业堆术语。

## 观点型

适用于对现象、事件或普遍看法作出判断。

推荐结构：

1. 点出争议、误区或现实冲突。
2. 明确说出用户的判断。
3. 给出两到三个最强理由，按因果而不是口号排列。
4. 回应最有力的反对意见，并说清观点边界。
5. 用一句可复述的话收束。

不要把立场写得比用户更极端。不要捏造“所有人都认为”一类假想对手。

## 教程型

适用于教观众完成一个具体动作或获得一个可验证结果。

推荐结构：

1. 先说完成后能得到什么，以及适合谁。
2. 说明必要前提或准备。
3. 按实际操作顺序讲步骤，每一步包含动作和判断标准。
4. 指出最容易失败的一处及修正方式。
5. 给出观众现在可以执行的最小下一步。

不要把工具清单冒充教程。步骤必须能被执行，不能只讲原则。

## 故事型

适用于用真实经历或已提供的案例承载一个观点，不包含虚构剧情对白。

推荐结构：

1. 从具体场景、目标或冲突开始。
2. 只保留推动变化的事件。
3. 展示关键转折或认知变化。
4. 解释这个故事为什么支持核心观点。
5. 回到观众处境，用观点收束。

不要虚构用户经历、人物心理或对话。缺少关键事实时先询问，不要用想象补齐。

## 产品或营销型

适用于介绍产品、服务、课程或付费方案，并推动明确行动。

推荐结构：

1. 说清目标用户正在经历的具体问题。
2. 给出产品承诺，但只使用可证实的范围。
3. 解释产品通过什么机制解决问题。
4. 使用用户提供的演示、案例、数据或可信事实作为证据。
5. 说明适合谁、不适合谁，以及明确的下一步行动。

不要虚构销量、评价、稀缺性、效果保证或权威背书。证据不足时降低承诺，不要制造证据。

## 混合题材

选择一个决定整体推进方式的主类型，再借用一个辅助类型：

- 知识 + 故事：用故事承载机制，但结论仍是知识解释。
- 观点 + 故事：用经历证明判断，但不要让故事盖过立场。
- 教程 + 产品：先让步骤成立，再自然说明产品在哪一步提供帮助。
- 知识 + 产品：先讲清问题机制，再说明产品为何适用。

不要在一篇短口播里同时使用三种以上结构。
